# Database Design: Delivery Service System

This project focuses on designing and implementing a robust relational database for a **Delivery Service** domain. It covers the full database development lifecycle, from conceptual modeling to physical implementation in **PostgreSQL**.

## 📌 Project Overview
The system is designed to manage core delivery operations, including customer data, order processing, courier assignments, and real-time delivery status tracking.

## 🛠 Tech Stack
* **Database:** PostgreSQL
* **IDE:** DataGrip
* **Modeling:** EERD (Conceptual) & RSD (Relational)

## 📂 Deliverables
1. **EERD (Enhanced Entity-Relationship Diagram):** Conceptual model defining key entities (Customers, Orders, Products, Couriers) and their business relationships.
2. **RSD (Relational Schema Diagram):** Logical model showcasing table structures, data types, Primary Keys, and Foreign Keys.
3. **Database Normalization:** The schema is normalized to **3NF** (Third Normal Form) to eliminate data redundancy and ensure integrity.

## 📈 Key Accomplishments (for Resume)
* Developed a relational database schema for a complex **Delivery Service** domain.
* Designed **EERD** and **RSD** diagrams to model business processes and ensure data integrity.
* Normalized the database structure to **3NF** to maintain data consistency and reduce redundancy.
* Implemented the physical schema in **PostgreSQL** using **DataGrip**, establishing primary keys, foreign keys, and indexes for optimized performance.
